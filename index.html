TUGAS BAB 1

OSPFv2 Concepts

Fitur dan Karakteristik OSPF

Open Shortest Path First (OSPF) adalah protokol routing link-state yang dikembangkan sebagai alternatif untuk distance vector Routing Information Protocol (RIP). OSPF memiliki keunggulan signifikan dibandingkan RIP karena menawarkan konvergensi yang lebih cepat dan skala implementasi jaringan yang jauh lebih besar. OSPF adalah protokol routing link-state yang menggunakan konsep area untuk skalabilitas. Router yang menjalankan OSPF bertukar pesan untuk menyampaikan informasi routing menggunakan lima jenis paket: paket Hello, paket deskripsi database, paket permintaan link-state, paket pembaruan link-state, dan paket pengakuan link-state. Pesan OSPF digunakan untuk membuat dan memelihara tiga database OSPF: database adjacency membuat tabel tetangga, database link-state (LSDB) membuat tabel topologi, dan database penerusan membuat tabel routing

Fitur dan Karakteristik OSPF

Router membangun tabel topologi menggunakan hasil perhitungan berdasarkan algoritma Dijkstra SPF (shortest-path first). Algoritme SPF didasarkan pada biaya kumulatif untuk mencapai suatu tujuan. Di OSPF, biaya digunakan untuk menentukan jalur terbaik menuju tujuan. Untuk menjaga informasi routing, router OSPF menyelesaikan proses routing link-state generik untuk mencapai keadaan konvergensi:

1. Membangun Kedekatan Tetangga

2. Iklan Exchange Link-State

3. Bangun Basis Data Link State

4. Jalankan Algoritma SPF

5. Pilih Rute Terbaik


Dengan OSPF area tunggal, nomor apa pun dapat digunakan untuk area tersebut, praktik terbaiknya adalah menggunakan area 0. OSPF area tunggal berguna dalam jaringan yang lebih kecil dengan sedikit router. Dengan OSPF multiarea, satu domain perutean besar dapat dibagi menjadi area yang lebih kecil, untuk mendukung perutean hierarki. Perutean masih terjadi antar area (perutean antar area), sementara banyak operasi perutean intensif prosesor, seperti penghitungan ulang database, disimpan dalam suatu area. OSPFv3 adalah setara OSPFv2 untuk menukar awalan IPv6. Ingatlah bahwa dalam IPv6, alamat jaringan disebut sebagai awalan dan subnet mask disebut panjang awalan.

Paket OSPF

OSPF menggunakan paket link-state (LSP) berikut untuk membangun dan memelihara kedekatan tetangga dan bertukar pembaruan perutean: 1 Hello, 2 DBD, 3 LSR, 4 LSU, dan 5 LSAck. LSU juga digunakan untuk meneruskan pembaruan perutean OSPF, seperti perubahan tautan. Paket hello digunakan untuk:

• Temukan tetangga OSPF dan buat kedekatan tetangga.

• Iklankan parameter di mana dua router harus setuju untuk menjadi tetangga.

• Pilih Designated Router (DR) dan Backup Designated Router (BDR) pada jaringan multiakses seperti Ethernet. Tautan point-to-point tidak memerlukan DR atau BDR.

Beberapa field penting dalam paket Hello adalah tipe, ID router, ID area, network mask, interval hello, prioritas router, interval mati, DR, BDR dan daftar tetangga.

Operasi OSPF

Status yang dilalui OSPF untuk melakukan hal ini adalah status turun, status init, status dua arah, status ExStart, status Exchange, status pemuatan, dan status penuh. Ketika OSPF diaktifkan pada sebuah antarmuka, router harus menentukan apakah ada tetangga OSPF lain pada link tersebut dengan mengirimkan paket Hello yang berisi ID routernya ke semua antarmuka yang mendukung OSPF. Paket Hello dikirim ke alamat multicast IPv4 Semua Router OSPF yang dicadangkan 224.0.0.5. Hanya router OSPFv2 yang akan memproses paket ini. Ketika router tetangga yang mendukung OSPF menerima paket Hello dengan ID router yang tidak ada dalam daftar tetangganya, router penerima mencoba untuk membangun kedekatan dengan router yang memulai. Setelah status Dua Arah, router bertransisi ke status sinkronisasi database, yang merupakan proses tiga langkah:

1. Tentukan Router Pertama

2. Tukarkan DBD

3. Kirim LSR

Jaringan multiakses dapat menciptakan dua tantangan bagi OSPF sehubungan dengan membanjirnya LSA: penciptaan beberapa kedekatan dan membanjirnya LSA secara ekstensif. Peningkatan dramatis dalam jumlah router juga secara dramatis meningkatkan jumlah LSA yang dipertukarkan antar router. Membanjirnya LSA ini berdampak signifikan terhadap pengoperasian OSPF. Jika setiap router di jaringan multiakses harus membanjiri dan mengakui semua LSA yang diterima ke semua router lain di jaringan multiakses yang sama, lalu lintas jaringan akan menjadi sangat kacau. Oleh karena itu pemilihan DR dan BDR perlu dilakukan. Pada jaringan multiakses, OSPF memilih DR untuk menjadi titik pengumpulan dan distribusi LSA yang dikirim dan diterima. BDR juga dipilih jika DR gagal.

TUGAS BAB 2

Configuration

ID Perute OSPF

OSPFv2 diaktifkan menggunakan perintah mode konfigurasi global router ospf process-id . Nilai id proses mewakili angka antara 1 dan 65.535 dan dipilih oleh administrator jaringan. ID router OSPF adalah nilai 32-bit, yang direpresentasikan sebagai alamat IPv4. ID router digunakan oleh router berkemampuan OSPF untuk menyinkronkan database OSPF dan berpartisipasi dalam pemilihan DR dan BDR. Router Cisco memperoleh ID router berdasarkan salah satu dari tiga kriteria, dalam urutan preferensi berikut:

1. ID router dikonfigurasikan secara eksplisit menggunakan perintah mode konfigurasi router-id router OSPF . Nilai rid adalah nilai 32-bit apa pun yang dinyatakan sebagai alamat IPv4.

2. Jika ID router tidak dikonfigurasi secara eksplisit, router memilih alamat IPv4 tertinggi dari semua antarmuka loopback yang dikonfigurasi.

3. Jika tidak ada antarmuka loopback yang dikonfigurasi, maka router memilih alamat IPv4 aktif tertinggi dari semua antarmuka fisiknya.

ID router dapat ditetapkan ke antarmuka loopback. Alamat IPv4 untuk antarmuka loopback jenis ini harus dikonfigurasi menggunakan subnet mask 32-bit (255.255.255.255), yang membuat rute host. Rute host 32-bit tidak akan diiklankan sebagai rute ke router OSPF lainnya. Setelah router memilih ID router, router OSPF yang aktif tidak mengizinkan ID router diubah hingga router dimuat ulang atau proses OSPF direset. Gunakan perintah proses clear ip ospf untuk mengatur ulang kedekatan. Anda kemudian dapat memverifikasi bahwa R1 menggunakan perintah ID router baru dengan perintah tampilkan protokol ip yang disalurkan untuk hanya menampilkan bagian ID router.

Jaringan OSPF Titik-ke-Titik

Perintah jaringan digunakan untuk menentukan antarmuka mana yang berpartisipasi dalam proses perutean untuk area OSPFv2. Sintaks dasar untuk perintah jaringan adalah network -address wildcard-mask area area-id. Setiap antarmuka pada router yang cocok dengan alamat jaringan dalam perintah jaringan dapat mengirim dan menerima paket OSPF. Saat mengonfigurasi OSPFv2 area tunggal, perintah jaringan harus dikonfigurasi dengan nilai id area yang sama di semua router. Wildcard mask biasanya merupakan kebalikan dari subnet mask yang dikonfigurasi pada antarmuka tersebut. Dalam topeng wildcard:

• Wildcard mask bit 0 - Cocok dengan nilai bit yang sesuai di alamat

• Wildcard mask bit 1 - Mengabaikan nilai bit yang sesuai di alamat

Dalam mode konfigurasi perutean, ada dua cara untuk mengidentifikasi antarmuka yang akan berpartisipasi dalam proses perutean OSPFv2. Salah satu caranya adalah ketika masker wildcard mengidentifikasi antarmuka berdasarkan alamat jaringan. Setiap antarmuka aktif yang dikonfigurasi dengan alamat IPv4 milik jaringan tersebut akan berpartisipasi dalam proses perutean OSPFv2. Cara lainnya adalah OSPFv2 dapat diaktifkan dengan menentukan alamat IPv4 antarmuka yang tepat menggunakan masker wildcard quad zero.


Perintah tampilkan protokol ip kemudian digunakan untuk memverifikasi bahwa antarmuka Loopback 0 terdaftar sebagai pasif. Proses pemilihan DR/BDR tidak diperlukan karena hanya ada dua router pada jaringan point-to-point antara R1 dan R2. Gunakan perintah konfigurasi antarmuka ip ospf network point-to-point pada semua antarmuka di mana Anda ingin menonaktifkan proses pemilihan DR/BDR. Gunakan loopback untuk mensimulasikan lebih banyak jaringan daripada yang dapat didukung oleh peralatan. Secara default, antarmuka loopback diiklankan sebagai rute host /32. Untuk mensimulasikan LAN nyata, antarmuka Loopback 0 dikonfigurasi sebagai jaringan point-to-point.

Ubah OSPF area tunggal

OSPF menggunakan biaya sebagai metrik. Biaya yang lebih rendah menunjukkan jalur yang lebih baik daripada biaya yang lebih tinggi. Biaya Cisco untuk sebuah antarmuka berbanding terbalik dengan bandwidth antarmuka. Oleh karena itu, bandwidth yang lebih tinggi menunjukkan biaya yang lebih rendah. Rumus yang digunakan untuk menghitung biaya OSPF adalah: Biaya = bandwidth referensi/bandwidth antarmuka. Karena nilai biaya OSPF harus berupa bilangan bulat, antarmuka FastEthernet, Gigabit Ethernet, dan 10 GigE berbagi biaya yang sama. Untuk memperbaiki situasi ini, Anda dapat menyesuaikan bandwidth referensi dengan perintah auto-cost reference-bandwidth pada setiap router OSPF, atau secara manual mengatur nilai biaya OSPF dengan perintah ip ospf cost . Untuk menyesuaikan bandwidth referensi, gunakan perintah konfigurasi router Mbps bandwidth referensi biaya otomatis . Biaya rute OSPF adalah nilai akumulasi dari satu router ke jaringan tujuan. Nilai biaya OSPF dapat dimanipulasi untuk mempengaruhi rute yang dipilih oleh OSPF. Untuk mengubah laporan nilai biaya oleh router OSPF lokal ke router OSPF lainnya, gunakan perintah konfigurasi antarmuka ip ospf cost value . Jika interval Mati berakhir sebelum router menerima paket Hello, OSPF menghapus tetangga tersebut dari database link-state (LSDB). Router membanjiri LSDB dengan informasi tentang tetangga bawah dari semua antarmuka yang mendukung OSPF. Cisco menggunakan default 4 kali interval Hello atau 40 detik pada jaringan multiakses dan point-to-point. Untuk memverifikasi interval antarmuka OSPFv2, gunakan perintah show ip ospf interface . Interval Halo dan Mati OSPFv2 dapat dimodifikasi secara manual menggunakan perintah mode konfigurasi antarmuka berikut: ip ospf hello-interval detik dan ip ospf detik interval mati .

Jenis Jaringan OSPF

Router dapat dihubungkan ke switch yang sama untuk membentuk jaringan multiakses. LAN Ethernet adalah contoh paling umum dari jaringan multiakses siaran. Dalam jaringan siaran, semua perangkat di jaringan melihat semua frame siaran dan multicast. DR bertanggung jawab untuk mengumpulkan dan mendistribusikan LSA. DR menggunakan alamat IPv4 multicast 224.0.0.5 yang ditujukan untuk semua router OSPF. Jika DR berhenti memproduksi paket Hello, BDR mempromosikan dirinya sendiri dan mengambil peran DR. Semua router lain menjadi DROTHER. DROTHER menggunakan alamat multiakses 224.0.0.6 (semua router yang ditunjuk) untuk mengirim paket OSPF ke DR dan BDR. Hanya DR dan BDR yang mendengarkan 224.0.0.6. Untuk memverifikasi peran router OSPFv2, gunakan perintah show ip ospf interface . Untuk memverifikasi kedekatan OSPFv2, gunakan perintah show ip ospf neighbour . Keadaan tetangga dalam jaringan multiakses dapat berupa:

• FULL/DROTHER - Ini adalah router DR atau BDR yang sepenuhnya berdekatan dengan router non-DR atau BDR.

• FULL/DR - Router sepenuhnya berdekatan dengan tetangga DR yang ditunjukkan.

• FULL/BDR - Router sepenuhnya berdekatan dengan tetangga BDR yang ditunjukkan.

• 2-WAY/DROTHER - Router non-DR atau BDR memiliki hubungan tetangga dengan router non-DR atau BDR lainnya.

Pemilihan OSPF DR dan BDR tidak bersifat pre-emptive. Jika DR gagal, BDR otomatis dipromosikan menjadi DR. Hal ini terjadi bahkan jika DROTHER lain dengan prioritas lebih tinggi atau ID router ditambahkan ke jaringan setelah pemilihan DR/BDR awal. Namun, setelah BDR dipromosikan menjadi DR, pemilihan BDR baru terjadi dan DROTHER dengan prioritas tertinggi atau ID router terpilih sebagai BDR baru. Untuk mengatur prioritas suatu antarmuka, gunakan perintah nilai prioritas ip ospf , dimana nilainya 0 hingga 255. Jika nilainya 0, router tidak akan menjadi DR atau BDR. Jika nilainya 1 hingga 255, maka router dengan nilai prioritas lebih tinggi kemungkinan besar akan menjadi DR atau BDR pada antarmuka.

Verifikasi OSPFv2 Area Tunggal

Dua perintah berikut digunakan untuk memverifikasi perutean:

• tampilkan antarmuka ip singkat – Digunakan untuk memverifikasi bahwa antarmuka yang diinginkan aktif dengan pengalamatan IP yang benar.

• tampilkan rute ip - Digunakan untuk memverifikasi bahwa tabel perutean berisi semua rute yang diharapkan.

Perintah tambahan untuk menentukan bahwa OSPF beroperasi seperti yang diharapkan meliputi: tampilkan ip ospf tetangga , tampilkan protokol ip , tampilkan ip ospf , dan tampilkan antarmuka ip ospf


Gunakan perintah show ip ospf neighbour untuk memverifikasi bahwa router telah membentuk kedekatan dengan router tetangganya. Untuk setiap tetangga, perintah ini menampilkan:

• Neighbor ID - ID router dari router tetangga.

• Pri - Prioritas antarmuka OSPFv2. Nilai ini digunakan pada pemilu DR dan BDR.

• Status - Status antarmuka OSPFv2. Status LENGKAP berarti router dan tetangganya memiliki LSDB OSPFv2 yang identik. Pada jaringan multiakses, seperti Ethernet, dua router yang berdekatan mungkin menampilkan statusnya sebagai 2WAY. Tanda hubung menunjukkan bahwa DR atau BDR tidak diperlukan karena jenis jaringan.

• Dead Time - Jumlah waktu yang tersisa dimana router menunggu untuk menerima paket OSPFv2 Hello dari tetangga sebelum mendeklarasikan tetangga tersebut down. Nilai ini direset ketika antarmuka menerima paket Hello.

• Alamat - Alamat IPv4 dari antarmuka tetangga yang terhubung langsung dengan router ini.

• Antarmuka - Antarmuka tempat router ini membentuk kedekatan dengan tetangganya.

Perintah tampilkan protokol ip adalah cara cepat untuk memverifikasi informasi konfigurasi OSPF penting seperti ID proses OSPFv2, ID router, antarmuka yang dikonfigurasi secara eksplisit untuk mengiklankan rute OSPF, tetangga tempat router menerima pembaruan, dan jarak administratif default, yang mana adalah 110 untuk OSPF. Gunakan perintah show ip ospf untuk memeriksa ID proses OSPFv2 dan ID router. Perintah ini menampilkan informasi area OSPFv2 dan terakhir kali algoritma SPF dijalankan. Perintah show ip ospf interface menyediakan daftar rinci untuk setiap antarmuka yang mendukung OSPFv2. Tentukan antarmuka hanya untuk satu antarmuka untuk menampilkan ID proses, ID router lokal, jenis jaringan, biaya OSPF, informasi DR dan BDR pada tautan multiakses, dan tetangga yang berdekatan.

Propagarasi Rute Default

Dalam terminologi OSPF, router yang terletak di antara domain routing OSPF dan jaringan non-OSPF disebut ASBR. Untuk menyebarkan rute default, ASBR harus dikonfigurasi dengan rute statis default menggunakan ip rute 0.0.0.0 0.0.0.0 [ next-hop-address | perintah exit-intf ] , dan perintah konfigurasi router asal informasi default . Ini menginstruksikan ASBR untuk menjadi sumber informasi rute default dan menyebarkan rute statis default dalam pembaruan OSPF. Verifikasi pengaturan rute default pada ASBR menggunakan perintah show ip rute .
